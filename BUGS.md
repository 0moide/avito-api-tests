# Баг-репорты


## BUG-001: Нестандартный формат ответа при создании объявления

| Поле | Значение |
|------|----------|
| **ID** | BUG-001 |
| **Краткое описание** | При успешном создании объявления API возвращает ID внутри строки `status`, а не в отдельном поле `id` |
| **Шаги воспроизведения** | 1. POST `/api/1/item` с валидными данными |
| **Фактический результат** | `{"status": "Сохранили объявление - 5cfac0f7-18e5-458b-8c90-1ce6b6bed3fe"}` |
| **Ожидаемый результат** | `{"id": "5cfac0f7-18e5-458b-8c90-1ce6b6bed3fe"}` |
| **Серьёзность** | Major |
| **Упавшие тесты** | `test_unique_item_ids`, `test_response_structure_create_item`, `test_create_item_success` |

## BUG-002: GET /statistic/{id} возвращает SSL ошибку для некорректного ID

| Поле | Значение |
|------|----------|
| **ID** | BUG-002 |
| **Краткое описание** | При запросе статистики с ID "invalid" сервер возвращает SSL ошибку вместо 400 |
| **Шаги воспроизведения** | 1. GET `/api/1/statistic/invalid` |
| **Фактический результат** | `SSLEOFError: EOF occurred in violation of protocol` |
| **Ожидаемый результат** | Статус 400 |
| **Серьёзность** | Critical |
| **Упавший тест** | `test_get_statistic_invalid_id_format` |

## BUG-003: GET /{sellerId}/item не валидирует sellerId

| Поле | Значение |
|------|----------|
| **ID** | BUG-003 |
| **Краткое описание** | GET запрос принимает невалидные sellerId (1, -1, abc) и возвращает 200 вместо 400 |
| **Шаги воспроизведения** | 1. GET `/api/1/1/item`<br>2. GET `/api/1/-1/item` |
| **Фактический результат** | Статус 200, пустой массив |
| **Ожидаемый результат** | Статус 400 |
| **Серьёзность** | Major |
| **Упавшие тесты** | `test_get_items_seller_id_out_of_range`, `test_get_items_negative_seller_id` |

## BUG-008: Невозможно проверить идемпотентность (следствие BUG-001)

| Поле | Значение |
|------|----------|
| **ID** | BUG-008 |
| **Краткое описание** | Из-за BUG-001 невозможно извлечь ID, поэтому нельзя проверить, что созданы разные объявления |
| **Шаги воспроизведения** | 1. Два одинаковых POST запроса |
| **Фактический результат** | ID не извлекаются, сравнение невозможно |
| **Ожидаемый результат** | Два разных ID |
| **Серьёзность** | Major |
| **Упавший тест** | `test_create_item_idempotency` |
